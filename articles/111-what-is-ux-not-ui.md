# UXとUIは何が違うのか——「見た目」ではなく「体験の総体」の正体

「このアプリ、UXがいいね」と言うとき、多くの人はきれいなボタンや配色を思い浮かべる。でもそれは正確には UI（ユーザーインターフェース）の話だ。UX（ユーザーエクスペリエンス）はもっと広い。**製品と出会う前から、使い終わった後まで、ユーザーが感じたことの全部**を指す。

有名なたとえがある。UXの提唱者ドン・ノーマンは、ある高級ホテルのレストランを例に出した。料理も内装も一流。でもトイレが分かりにくい場所にあり、迷って恥をかいた。すると「あの店は良かった」という記憶が濁る。**料理（UI）がどれだけ美しくても、体験全体（UX）は台無しになりうる**、というわけだ。

もう一つ身近な例を。ピザを注文するとする。UIは注文画面のボタンやメニューの見やすさ。でもUXは、「注文が本当に通ったか不安になる待ち時間」「配達予定が5分ずれて届くイライラ」「箱を開けたときの湯気」「次も頼みたくなる後味」まで含む。画面の外側にこそ、体験の大部分がある。

ここで面白いのは、**UXは必ずしもデジタルとは限らない**という点だ。ノーマンがこの言葉を広めた1990年代、彼はアップルで「ユーザーエクスペリエンス・アーキテクト」を名乗った。狙いは「人間工学やUIといった狭い言葉では、人がその製品で経験するすべてをカバーできない」という問題意識だった。パッケージを開ける所作、電源を入れる瞬間の待ち時間、困ったときのサポート——全部がUXだ。

実務では、この違いを意識しないと事故が起きる。たとえばECサイトで、商品ページ（UI）は美しいのに、購入完了までに入力欄が20個ある。統計的にも、**フォームの項目を減らすだけでコンバージョンが数十パーセント改善した**という事例は珍しくない。UIは満点でも、UXは赤点、ということが平気で起こる。

覚えておきたいのは、UIはUXの「一部」だということ。UIは体験に触れる接点（タッチポイント）の一つにすぎない。良いUXを設計する人は、画面の外——待ち時間、感情、前後の文脈——まで見ている。「見た目を整える」で止まらず、「その人の一日の中でこの体験はどう感じられるか」まで想像する。それがUXの視点だ。

次に何かアプリを触るとき、ぜひ問うてみてほしい。「これはUIが良いのか、それともUXが良いのか」。見分けられるようになると、世界の設計の質が急に見えてくる。

## 💡 アプリアイデアの種

- 案1: 自分の使ったサービスを「UI（見た目）」と「UX（体験全体）」の2軸で採点し、ギャップを可視化するレビュー記録アプリ。
- 案2: 注文〜受け取りまでの「不安になる待ち時間」を埋める、進捗をリアルタイムで見せる汎用トラッキング部品ライブラリ。
- 案3: 開発者向けに、自作アプリの「画面外の体験」（通知・待機・エラー後の導線）を洗い出すチェックリスト生成ツール。
