# コアループ：行動→報酬→成長の輪が面白さの心臓

なぜ人は「あと1回だけ」と言いながら深夜までゲームを続けてしまうのか。意志が弱いからではない。よくできたゲームには、プレイヤーを回し続ける「輪」が埋め込まれているからだ。この輪を**コアループ**と呼ぶ。

コアループの基本形はシンプルだ。**行動する → 報酬を得る → 強くなる（できることが増える）→ また行動する**。この4拍子がぐるぐる回る。輪が一周するたびにプレイヤーは少しだけ豊かになり、その豊かさが次の一周をより楽しくする。回るほど加速する、螺旋階段のような構造だ。

## ディアブロの「30秒の輪」

この構造を極端なまでに磨いたのが、ハック&スラッシュの名作『ディアブロ』シリーズだ。プレイヤーがやることは、突き詰めると「モンスターを倒す → アイテムが落ちる → 装備が強くなる → もっと強い敵を倒せる」の繰り返しにすぎない。開発元は「面白いコアループは30秒で一周する」ことを重視したという。数秒ごとに敵が倒れ、床がキラキラ光る戦利品で埋まる。その小さな報酬の連続が、気づけば何時間も指を止めさせない。

モバイルゲームはこの輪をさらに短く、そして巧妙に設計する。『Candy Crush』なら「盤面を消す→スコアとキラキラ演出→次のステージ」。1回のプレイが数分、報酬は即座。だからスキマ時間に開き、開けばまた輪に乗ってしまう。

## 「成長」が抜けると輪は失速する

コアループで最も見落とされやすいのが「成長」の要素だ。行動して報酬をもらうだけなら、それは同じことの単調な反復になる。飽きる。輪を回し続けさせるカギは、一周ごとにプレイヤー自身の**できることが増える**感覚——キャラが強くなる、新しい道具が解放される、あるいはプレイヤー自身が上手くなる——にある。この「成長」があるから、次の一周は前の一周と違う景色に見える。

開発現場では、企画段階でまず「このゲームの一番小さな輪は何秒で回るか」「一周ごとに何が増えるか」を紙に描く。輪がスカスカだと、どれだけ豪華なグラフィックを乗せても手が止まる。逆に輪さえ気持ちよく回れば、素朴な見た目でも人は夢中になる。『Vampire Survivors』は数百円の粗い見た目のゲームだが、「動くだけで敵を薙ぎ倒し→経験値→強化」の輪が中毒的で世界的ヒットになった。

面白さは足し算の派手さではなく、掛け算で回る輪の設計にある。あなたのアプリにも、ユーザーを気持ちよく一周させる「輪」はあるだろうか。

## 💡 アプリアイデアの種

- 学習アプリの「コアループ可視化」機能。問題を解く→即フィードバック→レベル解放、の輪が何秒で回るかを計測し、途中で離脱が起きる箇所を検出する。
- 習慣化アプリを「行動→小さな報酬→レベルアップ」のコアループ設計にする。1日1回の記録で必ず何かが増える（バッジ・成長する木・解放される機能）仕組み。
- 自作ゲームのコアループを1画面のフローチャートで設計・共有できるツール。「報酬」と「成長」の要素が欠けていると赤で警告する。
- 業務アプリにコアループ思考を持ち込み、日々の定型作業に「達成→可視化された成果→次の解放」を組み込んでモチベーションを維持させる。
